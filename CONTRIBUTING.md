# Contributing to OCIDex

## Prerequisites

OCIDex uses [Flox](https://flox.dev/) to manage its development environment. Flox provides Go, Node.js, npm, oras, syft, and other tools in a reproducible shell.

You'll also need:

- **PostgreSQL 15+** — via Docker Compose (`docker compose up -d postgres`) or a local install
- **Go 1.25+** and **Node 22+** — provided by Flox, but listed here for clarity

## Dev Setup

```sh
git clone https://github.com/pfenerty/ocidex.git
cd ocidex
flox activate

# Install Go tools (sqlc, golangci-lint)
make init

# Start PostgreSQL
docker compose up -d postgres

# Configure environment
cp .env.example .env   # edit DATABASE_URL if needed

# Run migrations
make migrate-up

# Verify everything works
make check        # fmt + lint + unit tests
make build         # compile the binary
make frontend      # build the SolidJS frontend
```

## Development Workflow

1. **Create a branch** from `main`.
2. **Make your changes.** See [Development Guide](docs/DEVELOPMENT.md) for coding patterns and examples.
3. **Run `make check`** before pushing — this runs formatting, linting, and unit tests.
4. **For database changes:** edit SQL in `db/queries/*.sql` or add a migration in `db/migrations/`, then run `make generate` and/or `make migrate-up`.
5. **For frontend changes:** run `make frontend-dev` for hot-reload development against the API on `:8080`.
6. **Open a pull request** with a clear description of what changed and why.

## Commit Conventions

This project uses [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add license category filtering
fix: handle null digest in SBOM summary
docs: update API overview in README
refactor: extract enrichment dispatcher options
test: add integration test for changelog endpoint
chore: update Go dependencies
```

## Running Tests

```sh
make test              # Unit tests (fast, no external deps)
make test-integration  # Integration tests (requires Docker for testcontainers)
make test-coverage     # Unit tests with HTML coverage report
```

## Code Generation

The repository layer is generated by sqlc. Never edit `internal/repository/*_sql.go` or `models.go` directly:

```sh
# Edit db/queries/*.sql, then:
make generate
```

The frontend TypeScript types are generated from the OpenAPI spec:

```sh
# After changing API handlers:
make openapi
```

## Architecture Context

Before proposing significant changes, review the [Architecture Decision Records](docs/adr/). These document the rationale behind every major technical choice. If your change would conflict with an existing ADR, open a discussion first — the ADR might need updating, or the change might need a different approach.
